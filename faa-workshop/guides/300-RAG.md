Your workshop environment also includes a pre-configured vector store that serves as the knowledge layer for this exercise. This instance contains the full-text contents of official FAA SWIM Operational Context Documents (CONOPS) found in the [sop](../sam/sop) directory.

This section will walk you through the creation of a native Solace Agent Mesh agent that leverages the RAG (Retrieval-Augmented Generation) [core plugin](https://github.com/SolaceLabs/solace-agent-mesh-core-plugins) to enable document ingestion and intelligent retrieval. The CONOPS Agent is a specialized RAG agent designed to provide intelligent, natural-language access to official FAA SWIM technical documentation without requiring users to manually search through hundreds of pages of dense systems engineering text. At its core, the agent performs semantic vector search across ingested documents, retrieving the most relevant passages in response to a question and synthesizing a precise, contextually grounded answer. For more information about the CONPS data, check out the [Introduction to FAA Data and Systems](./101-FaaData.md) document

For this tutorial, We will be using [Qdrant](https://qdrant.tech/) as the vector database of choice. 

## Configure your vector database

For the purpose of this workshop, we have configured cloud hosted Qdrant vector database to be used for the RAG agent

If running on your own, you have two options to configure your vector database

### 1. SaaS cloud managed

1. Navigate to [https://cloud.qdrant.io/](https://cloud.qdrant.io/) and sign up for an account
1. Create a vDB cluster
1. Copy the endpoint URL and Access Token
1. Configure your credentials in the [.env](../sam/.env) file respectively

### 2. Local Docker image
1. Follow the [Getting started with docker](https://qdrant.tech/documentation/quickstart/) page on the Qdrant documentation
1. Spin up your qdrant vDB docker image
1. Configure your credentials in the [.env](../sam/.env) file respectively

> Note: If you decide to user another third party vector database you can also do that! Read the [RAG documentation](https://solacelabs.github.io/solace-agent-mesh/docs/documentation/developing/tutorials/rag-integration) and the [Solace Agent Mesh core plugins](https://github.com/SolaceLabs/solace-agent-mesh-core-plugins) github repository

## Configure your RAG Agent

1. Kill the Solace Agent Mesh instance if its already running
    ```
    CTR + C
    ```

1. Execute the following from your `sam` dir
    ```
    cp ../solution/configs/agents/sop-agent.yaml configs/agents/sop-agent.yaml
    ```
1. Update the following two env variables in your [.env](../sam/.env)

    ```
    QDRANT_URL
    QDRANT_API_KEY
    ```
1. Run solace agent mesh
    ```
    sam run
    ```
1. [Optional] Open the [sop-agent.yaml](../sam/configs/agents/sop-agent.yaml) config file and explore the different sections
